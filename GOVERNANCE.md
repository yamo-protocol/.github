# YAMO Governance Model

YAMO is an open protocol governed by a lightweight, transparent process inspired by IETF and EIP workflows.

## Roles

### Core Maintainers
Responsible for:
- Approving PRs
- Managing releases
- Maintaining protocol integrity

### RFC Editors
Responsible for:
- Reviewing RFC proposals
- Ensuring consistency and clarity
- Advancing RFCs through the lifecycle

### Contributors
Anyone submitting issues, PRs, or RFCs.

## Decision Making
- Technical decisions are made through RFCs
- Implementation decisions are made through PR review
- Consensus is preferred; maintainers have final authority

## RFC Lifecycle
1. **Draft**
2. **Review**
3. **Final**
4. **Active / Deprecated**

## Transparency
All discussions occur in:
- GitHub Issues
- GitHub Discussions
- RFC PR threads
